# Modbus-Anomaly-Detector-ICS
**中国高校计算机大赛-A-ST网络空间与信息安全赛项** 参赛作品——工业控制系统Modbus协议异常检测系统，基于**结构体流水线**实现多人协作开发，融合**网络抓包、协议解析、机器学习异常检测、自动化防护**，贴合赛项**工业互联网安全、态势感知、智能与前沿安全**核心要求。

## 项目背景
工业控制系统（ICS）中Modbus协议因无加密、无认证成为网络攻击重灾区，本项目针对Modbus协议设计轻量级入侵检测系统（IDS），实现**实时抓包-特征提取-AI检测-自动化防护**的端到端解决方案，为工业互联网提供安全监测能力。

## 赛项贴合点
- 基础与通用安全：贴合**工业互联网安全**范畴
- 智能与前沿安全：融入**机器学习**进行异常检测
- 体系与应用场景：实现**态势感知、安全监测、自动化威胁猎捕**

## 团队分工（6人）
严格按流水线拆分，无代码耦合，每人独立负责一个模块：
| 组员 | 模块文件       | 核心职责                     | 技术栈               |
|------|----------------|------------------------------|----------------------|
| A    | src/capture.c  | 网络抓包，提取Modbus原始数据 | libpcap、网络协议    |
| B    | src/protocol.c | 协议解析，提取安全特征       | Modbus协议、数据统计 |
| C    | src/ai.c       | AI异常检测，判定异常类型     | 机器学习、均值方差   |
| D    | src/protect.c  | 自动化防护，执行阻断/告警    | iptables、系统日志   |
| E    | src/common.c   | 公共工具，日志/时间戳等      | C标准库、系统调用    |
| F    | src/web.c      | 网页展示，实时推送检测结果   | JSON、前端轮询       |

## 工程结构
Modbus-Anomaly-Detector-ICS/  # 仓库根目录
├── include/                  # 公共头文件（所有人共用，禁止随意修改）
│   └── modbus_def.h          # 统一数据结构（核心交接标准）
├── src/                      # 源代码目录（按分工拆分，每人一个文件）
│   ├── main.c                # 流水线主程序（总控，你负责维护）
│   ├── capture.c             # 组员A：抓包模块
│   ├── protocol.c            # 组员B：协议解析+特征提取
│   ├── ai.c                  # 组员C：AI异常检测
│   ├── protect.c             # 组员D：防护执行
│   ├── common.c              # 公共工具函数（如时间戳、日志，可选1人负责）
│   └── web.c                 # 拓展：网页状态展示（可选1人负责，6人分工补位）
├── Makefile                  # 编译脚本（一键编译，无需手动敲命令）
├── README.md                 # 项目说明（赛项申报/队友开发参考）
├── .gitignore                # GitHub忽略文件（不提交编译产物/临时文件）
└── test/                     # 测试目录（存放测试用Modbus数据包，可选）
    └── test_packet.dat       # 测试数据包示例
